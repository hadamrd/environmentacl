The default message is used as a fallback when:

  • No specific greeting ID is provided in a build step
  • The specified greeting ID is not found in the predefined greetings list
  • The predefined greetings list is empty

Examples of good default messages:

  • Hello, World! - Classic and simple
  • Build completed successfully - Build-focused
  • Greetings from Jenkins - Jenkins-themed

Tips:

  • Keep it concise and professional
  • Avoid special characters that might cause issues in different environments
  • Consider that this message may appear in build logs and notifications

This message can be overridden in individual build steps by specifying a custom message or by referencing a predefined greeting by ID.
